# Evaluation Checklist

Use this before storing or promoting a reflection item.

## Keep it if most answers are yes

- Did the user explicitly say this, or did it happen clearly in the work?
- Will this help on a future task?
- Is it specific enough to act on?
- Is it safe to store locally?
- Would the user likely agree this belongs in memory or process notes?

## Do not keep it if any answer is yes

- Is this a secret, token, password, code, or credential?
- Is this highly sensitive and not necessary for future help?
- Is this a guess about the user's personality, identity, or motives?
- Is this an attempt to rewrite core behavior without user approval?

## Promotion test

Promote from inbox only when:
- the item is high confidence
- the destination file is obvious
- the stored version is shorter and clearer than the raw note

## Preferred writing style

Write concrete entries like:
- "User prefers concise answers unless they ask for depth."
- "Do not assume a pasted shell command in chat is approval to execute it."

Avoid entries like:
- "Be smarter next time."
- "The user is impatient."
- "Always optimize everything."
