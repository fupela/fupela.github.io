## IMPORTANT: How jedaiflow.com is Served

jedaiflow.com is served by Cloudflare Pages via **direct upload**, NOT from GitHub Pages or a git-connected CF Pages project.

- Pushing to `fupela/fupela.github.io` on GitHub does **NOT** update jedaiflow.com
- This repo is a backup / source of truth — it is not the deploy trigger
- To update jedaiflow.com, you must run `wrangler pages deploy` (see below)

---

## Deploy Command (Fupie — Run on Mac Mini)

From the workspace directory `/Users/fupie/.openclaw/workspace/projects/fupela.github.io/`:

```bash
cd /Users/fupie/.openclaw/workspace/projects/fupela.github.io/

# Step 1: Deploy to Cloudflare Pages
npx wrangler pages deploy . --project-name jedaiflow

# Step 2: Verify deployment (wait ~30s then curl)
curl -sI https://jedaiflow.com | head -5
```

> CF API credentials are required (wrangler must be authenticated). On the Mac Mini, run `npx wrangler whoami` to verify. If not logged in: `npx wrangler login` or set `CLOUDFLARE_API_TOKEN` in environment.

## Option: Connect GitHub to CF Pages (one-time setup)

To enable auto-deploy on git push:

1. Go to https://dash.cloudflare.com → Workers & Pages
2. Find the `jedaiflow` project → Settings → Git integration
3. Connect to `fupela/fupela.github.io` repository
4. Build settings: **No build step** (static HTML)
5. Deploy directory: `/` (root)

After setup, every `git push` auto-deploys to jedaiflow.com.
